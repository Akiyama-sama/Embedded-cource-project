# 银行营业大厅排队叫号系统

这是一个银行大厅排队叫号系统。当前版本的推荐部署方式是：

- 你自己的电脑运行服务端
- 1 块开发板作为取号端
- 1 块开发板作为叫号窗口端
- 三者通过局域网使用 `TCP/IP` 通信

## 部署/通信概览

- 服务端运行在电脑上，使用 `FastAPI`
- 取号端和叫号端都是独立客户端页面，可以分别在两块板子上打开
- 页面请求使用 `HTTP`，实时同步使用 `WebSocket`
- `HTTP` 和 `WebSocket` 都建立在 `TCP/IP` 之上，所以通信方式满足“使用 TCP/IP 协议”的要求

如果你现场只演示 1 个叫号窗口，那么只需要打开 `station-1` 这一页；`station-2` 只是保留给扩展演示。

## 系统结构

- `电脑服务端`：负责发号、排队调度、SQLite 持久化、实时广播
- `取号端开发板`：打开取号页面，客户点击按钮取号
- `叫号端开发板`：打开叫号页面，窗口点击按钮呼叫下一位或重复叫号

## 功能

- 两种号码类型：`vip` 和 `normal`
- 叫号规则：先按优先级，再按等待时间
- 当日号票有效，跨天自动失效
- SQLite 持久化，服务重启后可继续叫号
- WebSocket 实时同步
- 叫号时浏览器语音播报

## 通信方式

- 取号端到服务端：`HTTP /api/tickets`
- 叫号端到服务端：`HTTP /api/stations/{station_id}/next` 和 `/repeat`
- 服务端到客户端状态同步：`WebSocket /ws`

这些都运行在局域网 `TCP/IP` 网络之上。

## 部署拓扑

- 电脑：运行 Python 服务端
- 开发板 A：作为取号端，浏览器打开 `/take-number`
- 开发板 B：作为叫号端，浏览器打开 `/station/station-1`
- 所有设备接入同一局域网，并能互相访问

## 环境要求

- 电脑端：Python 3.11+
- 客户端开发板：能联网并可运行 Chromium 类浏览器
- 同一局域网

## 安装

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'
```

## 启动服务端

在你自己的电脑上运行：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

启动后，先确认电脑的局域网 IP，例如 `192.168.1.10`。

## 客户端访问地址

假设电脑 IP 为 `192.168.1.10`：

- 取号端开发板打开：`http://192.168.1.10:8000/take-number`
- 叫号端开发板打开：`http://192.168.1.10:8000/station/station-1`

如果需要扩展多个窗口，还可以继续使用：

- `http://192.168.1.10:8000/station/station-2`
- `http://192.168.1.10:8000/station/station-3`

## 环境变量

- `QUEUE_DB_PATH`：SQLite 文件路径，默认 `data/queue.sqlite3`
- `CALL_STATION_COUNT`：窗口数量，默认 `2`
- `BUSINESS_TZ`：营业日时区，默认 `Asia/Shanghai`

## 演示流程

1. 电脑启动服务端
2. 取号端开发板打开取号页面
3. 叫号端开发板打开 `station-1` 页面
4. 在取号端点击“优先业务”或“普通业务”
5. 在叫号端点击“呼叫下一位”
6. 页面显示当前叫号结果，并播放语音

## 数据与恢复

- 未叫号的票号会写入 SQLite
- 服务端重启后读取同一数据库即可继续服务
- 跨天后，前一天未叫到的等待号会自动标记为失效

## 常见问题

### 板子打不开页面

确认电脑和开发板在同一局域网，并且访问的是电脑的局域网 IP，不是 `127.0.0.1`。

### 浏览器不播报

确认开发板浏览器允许语音合成。当前叫号播报使用浏览器 `speechSynthesis`。

### 数据库文件在哪

默认在服务端电脑的 `data/queue.sqlite3`。
